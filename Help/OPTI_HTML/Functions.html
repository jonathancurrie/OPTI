Introduction

The OPTI Toolbox provides one main class: opti. The opti class provides functionality for building, solving and analyzing optimization problems within MATLAB, and while fully optional (you can use the supplied solvers directly if you prefer), it does provide a level of convenience and error checking that can make solving optimization problems more reliable.

As the OPTI Toolbox has matured, more of the documentation has moved to the OPTI Wiki, available online. This should be your main source of documentation. In addition, the Examples Folder included with OPTI contains many examples on how to use the supplied functionality. It is noted however that not all low level functionality is covered in these locations, so it is briefly covered below.

Getting Help with OPTI Functions

All OPTI functions, whether part of the opti class or not, are documented. This means you can type:

>> help optiprob

To obtain help for a specific function. If the function is a method of a class, you can type:

>> help opti/solve

which will return information on that method.

 

OPTI Class Methods

opti [constructor]

The opti constructor is a flexible constructor that can accept any number of arguments, using the 'parameter', 'value' pair format. It is used to create an optimization problem in a form suitable for the selected solver. Use

>> help opti

for more information. In addition, to view all compatible argument pairs, use

>> opti

which will print all argument options to the command window.

solve

Once an OPTI problem has been created using the constructor above, it can be solved using the solve method. Use

>> help opti/solve

for more information.

checkSol

Once an OPTI problem has been solved you can verify the solution return is infact feasible given the original constraints using the checkSol method. Use

>> help opti/checkSol

for more information.

plot

Once an OPTI problem has been solved (or if bounded, it doesn't have to be solved) you can plot the optimization objective and constraint contours using plot. The method works best for 2D problems, but also works for 1D->5D problems. Use

>> help opti/plot

for more information.

fitStats

When solving parameter estimation problems (such as NLS, DNLS) OPTI can calculate the fit statistics (e.g. confidence intervals, R-squared) associated with the solution. Once the problem has been solved, use fitStats to calculate the statistics. In addition, if you call plot after calling fitStats, it will plot (if available) the confidence bounds on the fitted function. Use

>> help opti/fitStats

for more information.

write

Once an OPTI problem has been entered using the constructor, it can then be written to a common optimization file format (such as MPS, SDPA, GAMS, etc) using the write method. Use

>> help opti/write

for more information.

multisolve

OPTI implements a very naive multi-start solver for applying local solvers to global optimization problems. Use multisolve to initiate the multi-start solver. Use

>> help opti/multisolve

for more information.

multiplot

Once an OPTI problem has been solved used multisolve, you can use multiplot to view the optimization problem contours overlayed with the regions searched by the multi-start solver. Use

>> help opti/multiplot

for more information.

 

Solvers & Settings

optiSolver

To see what solvers are available in your distribution of OPTI, you can use the optiSolver function:

>> optiSolver

To see which solvers solve which optimization problems:

>> optiSolver('matrix')

To see which solvers solve a particular optimization problem:

>> optiSolver('prob_type')

where 'prob_type' is a standard optimization type. For example, for a Linear Program (LP):

>> optiSolver('lp')

To see which solvers support which optimization options (optiset below):

>> optiSolver('config')

For information on a particular solver use:

>> optiSolver('solver')

where 'solver' is the name of the solver you want information on. For example for CLP:

>> optiSolver('clp')

optiset

To create an options structure suitable for use with the opti constructor, use optiset. It follows the same argument format as the opti constructor (pairs). For more information on using optiset use:

>> help optiset

To print all available options and their defaults, use:

>> optiset

Visit the optiset Wiki page for more information.

solverset

Most solvers support many more options than are listed in optiset. Optiset is provided to simplify using common options, while for solver specific options, you need to use the solverset routines. The naming strategy is as follows: [solver_name + set], i.e. options for Nomad are set via nomadset, and options for Ipopt and set via ipoptset. To see how to use a solverset method, use

>> help solverset

for example for Ipopt:

>> help ipoptset

To see all available options for a solver, use

>> solverset

for example for Ipopt:

>> ipoptset

To see which solvers have extra solver settings, use

>> optiSolver('config')

and examine the sOpts column. Solvers with a x will  have a respective solverset method. Solver Options are supplied via the 'solverOpts' field to optiset.
