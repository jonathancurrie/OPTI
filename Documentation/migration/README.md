# Migration records

`page-map.json` defines the documentation hierarchy. `manifest.json` is
generated by the migration tool and records every source page, destination
file, route, content count, warning, and deliberate link exception.

Only each current public page body and assets from the source `Images`
directory are eligible for export. Deleted revisions, draft entries,
comments, PmWiki runtime/configuration files, access controls, credentials,
logs, and private metadata are excluded.

Published BlogIt news entries and the obsolete placeholder, sandbox, and
duplicate advanced-topic pages are deliberately excluded by `page-map.json`.

The two copied images `eq_milp.png` and `plot_ex1nls.png` are not referenced by
the current page bodies. They remain in the public asset set so that direct
legacy image URLs can continue to be served.
